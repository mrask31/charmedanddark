# Shopify Admin Configuration Guide

This document provides step-by-step instructions for configuring Shopify admin settings required for Phase 2 Revenue Mechanics.

## Task 9.1: Create HOUSE10 Discount Code

### Steps:

1. Log in to your Shopify Admin at https://admin.shopify.com/
2. Navigate to **Discounts** in the left sidebar
3. Click **Create discount** button
4. Select **Discount code**
5. Configure the discount with these settings:

   **Discount code**: `HOUSE10`
   
   **Type**: Percentage
   
   **Value**: 10%
   
   **Applies to**: All products
   
   **Minimum requirements**: None
   
   **Customer eligibility**: Everyone
   
   **Usage limits**:
   - ☐ Limit number of times this discount can be used in total (leave unchecked)
   - ☐ Limit to one use per customer (leave unchecked)
   
   **Active dates**:
   - Start date: Today
   - End date: No end date (leave blank)

6. Click **Save discount**

### Verification:

- The discount code HOUSE10 should appear in your Discounts list
- Status should show as "Active"
- You can test it by creating a test cart with the code applied

---

## Task 9.2: Configure Order Creation Webhook

### Steps:

1. In Shopify Admin, navigate to **Settings** (bottom left)
2. Click **Notifications**
3. Scroll down to the **Webhooks** section
4. Click **Create webhook**
5. Configure the webhook with these settings:

   **Event**: Order creation
   
   **Format**: JSON
   
   **URL**: `https://charmedanddark.com/api/webhooks/shopify/orders`
   
   **Webhook API version**: Latest (2024-01 or newer)

6. Click **Save webhook**

### Important Notes:

- The webhook URL must point to your **production domain**, not localhost
- Shopify will automatically add the `x-shopify-hmac-sha256` header for signature verification
- The webhook secret is automatically generated by Shopify

### Get Webhook Secret:

After creating the webhook, you need to get the webhook secret for HMAC verification:

1. In the webhook details page, look for the **Webhook signing secret**
2. Copy this value
3. Add it to your environment variables:
   - Development: Add to `.env.local` as `SHOPIFY_WEBHOOK_SECRET=your-secret-here`
   - Production: Add to Vercel environment variables as `SHOPIFY_WEBHOOK_SECRET`

---

## Task 9.3: Test Webhook Delivery

### Steps:

1. Ensure your application is deployed to production with the webhook handler
2. In Shopify Admin, go to **Settings** → **Notifications** → **Webhooks**
3. Find your Order creation webhook
4. Click on it to view details
5. Scroll down to **Recent deliveries** section
6. Create a test order in Shopify:
   - Go to **Orders** → **Create order**
   - Add a product
   - Add customer information
   - Mark as paid
   - Click **Create order**
7. Return to the webhook details page
8. Check **Recent deliveries** - you should see a new delivery with status 200 OK
9. Verify the order was stored in your Supabase database:
   - Open Supabase dashboard
   - Go to Table Editor → orders table
   - Look for the test order by shopify_order_id

### Troubleshooting:

If webhook delivery fails:

- **401 Unauthorized**: HMAC verification failed
  - Check that SHOPIFY_WEBHOOK_SECRET matches the secret from Shopify
  - Verify the webhook handler is using crypto.timingSafeEqual for comparison
  
- **500 Internal Server Error**: Application error
  - Check Vercel logs for error details
  - Verify Supabase credentials are correct
  - Ensure orders table exists with correct schema

- **Timeout**: Webhook handler took too long
  - Check for slow database queries
  - Verify Supabase connection is working

## Environment Variables Checklist

Before testing, ensure these environment variables are set in production (Vercel):

- [ ] `SHOPIFY_WEBHOOK_SECRET` - From Shopify webhook configuration
- [ ] `SHOPIFY_STORE_DOMAIN` - Your store domain (e.g., charmed-dark.myshopify.com)
- [ ] `SHOPIFY_STOREFRONT_ACCESS_TOKEN` - Storefront API token
- [ ] `NEXT_PUBLIC_SHOPIFY_CHECKOUT_DOMAIN` - Same as store domain
- [ ] `SUPABASE_SERVICE_ROLE_KEY` - For order storage
- [ ] `ANTHROPIC_API_KEY` - For lore generation (optional for checkout)
- [ ] `SYNC_API_SECRET` - For sync pipeline authentication

## Testing the Complete Flow

After configuration is complete, test the end-to-end flow:

1. Visit your production site
2. Browse to a product page
3. Click "Add to Selection"
4. Verify cart opens with correct pricing
5. Click "Proceed to Checkout"
6. Verify redirect to Shopify checkout
7. Verify HOUSE10 discount is automatically applied (10% off)
8. Complete the test purchase
9. Check Supabase orders table for the new order
10. Verify webhook delivery in Shopify admin

## SQL Migration Required

Before deploying, run this SQL in Supabase SQL Editor:

```sql
-- Add shopify_variant_id column if it doesn't exist
ALTER TABLE products 
ADD COLUMN IF NOT EXISTS shopify_variant_id TEXT;

-- Create index for fast lookups
CREATE INDEX IF NOT EXISTS idx_products_shopify_variant_id 
ON products(shopify_variant_id);
```

This migration is saved in `.kiro/specs/phase2-revenue-mechanics/migrations/001_add_shopify_variant_id.sql`
